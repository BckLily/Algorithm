
Bulle Sort(거품 정렬)
1. 앞에서부터 2개씩 데이터를 묶어서 값을 비교하고 둘 중 더 큰 수가 뒤로가도록 자리를 바꾼다.
2. 다음 2개의 데이터들도 마찬가지로 비교하여 둘중 더 큰 수가 뒤로가게 한다.
3. 이를 마지막 데이터까지 반복한다.
4. 맨 마지막 데이터 1개가 정렬이 이루어진다.
5. 다시 처음부터 반복하되 이미 정렬된 맨 뒤의 데이터는 제외한다.
6. 이를 반복하면 뒤에서부터 데이터가 정렬한다.
7. 정렬이 진행될수록 반복하는 (비교하는) 횟수가 줄어든다.


SelectionSort(선택 정렬)
1. 맨 앞 데이터부터 맨 마지막 데이터까지 확인하면서 가장 작은 데이터가 있는 위치값을 저장한다.
2. 최종적으로 저장된 위치값에 저장
3. 데이터를 맨 앞의 데이터와 자리를 바꿔준다.
4. 맨 앞의 데이터가 정렬됐기 때문에 나머지 데이터를 가지고 마찬가지로 가장 작은 데이터가 저장된 위치를 찾는다.
5. 찾아낸 위치의 데이터를 남아있는 데이터 중에서 가장 앞으로 가져온다.
6. 반복



InsertionSort(삽입 정렬)
1. 삽입 정렬은 첫 번째 데이터가 아니라 두 번째 데이터부터 시작한다.
2. 자신의 바로 왼쪽에 있는 데이터와 크기를 비교해서 왼쪽이 더 크면 서로 자리를 바꾼다.
3. 만약 자리가 바뀌었으면 바뀐 자리의 왼쪽 데이터와 자기 자신의 크기를 비교한다. 왼쪽이 크면 자리를 바꾼다.
4. 제일 왼쪽에 해당 데이터가 도착하거나 왼쪽 데이터가 자신보다 크지 않을 때까지 반복한다.
5. 마지막 데이터까지 반복.


선택 정렬 및 버블 정렬은 해당 위치에 들어갈 데이터를 찾는게 목적
삽입 정렬은 해당 데이터가 들어갈 위치를 찾는것이 목적.

선택 정렬과 버블 정렬은 모든 데이터를 항상 비교하기 때문에
시간이 오래 걸리고 항상 정렬에 걸리는 시간이 동일한 반면

삽입 정렬은 자신의 왼쪽 데이터만 비교하고 왼쪽 데이터는 이미 정렬된 데이터이기 때문에
비교 도중에 나머지 전체를 비교하지 않고 다음 정렬로 넘어갈 수 있다.
따라서 저장된 데이터가 정렬되어 있을수록 정렬에 걸리는 시간이 적다.



퀵 정렬(Quick Sort)
기준이 되는 값을 하나 정하고
기준갑보다 작은 값을 기준의 왼쪽에
기준값보다 큰 값을 기준의 오른쪽에 배치하고
각각 기준값보다 작은 값들을 가지고
퀵 정렬을 하고
기준값보다 큰 값들을 가지고 다시 퀵 정렬을 하는 것을 반복

1. 퀵정렬을 할 범위를 정한다. (start, end) 배열의 번호
2. 분류의 기준이 되는 pivot을 정한다. (start와 같은 값을 둔다)
3. 분류를 하기 위한 Big과 Small을 정한다. (Big은 pivot보다 큰 값의 위치를 찾기위한 변수, Small은 Pivot보다 작은 값의 위치를 찾기 위한 변수)
4. Big은 piovt바로 오른쪽에서 시작하고 Small은 End에서 시작한다.
5. Big은 Pivot보다 큰 값을 찾기 위해서 오른쪽으로 이동 Small은 Pivot보다 작은 값을 찾기 위해서 오른쪽으로 이동
6. 각각의 변수가 원하는 값이 있는 위치를 찾으면 멈춘다.
7. Big과 Small이 원하는 값을 찾았으면 Big과 Small에 있는 데이터를 바꿔준다.
8. 데이터를 바꿔준 후에 다시 Big과 Small은 원하는 값을 찾기 위해서 이동한다.
9. 데이터를 찾으면 서로 데이터를 바꾸고 반복
10. Big과 Small이 위치를 찾는 과정에서 서로의 위치가 바뀌면(Big이 Small보다 커지거나 Small이 Big보다 작아지면)
    위치 탐색을 종료하고 Pivot과 Small의 데이터를 바꾼다.
11. Pivot과 Small의 위치가 바뀐 시점에서 퀵 정렬의 한 사이클이 끝나고
    이동된 Pivot의 왼쪽에는 Pivot보다 작은 값 오른쪽에는 Pivot보다 큰 값이 정렬된다.
    (순서는 상관없이 Pivot보다 큰지 작은지로만 정렬된다.)
12. Pivot의 왼쪽에 있는 데이터들로 다시 퀵 정렬을 시작하고
    Pivot의 오른쪽에 있는 데이터로 다시 퀵정렬을 시작한다.
13. Pivot의 왼쪽의 데이터는 start는 기존의 pivot의 start와 동일하고
    end는 이동한 pivot의 왼쪽이 된다.
14. Pivot의 오른쪽에 있는 데이터는 Start는 pivot의 오른쪽이고
    end는 end이다.
15. Start와 End가 같거나 Start가 End보다 크면
    정렬할 숫자가 하나밖에 없다는 뜻이기 때문에 별도의 정렬을 하지 않는다.
